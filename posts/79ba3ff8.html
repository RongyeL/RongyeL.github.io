握手机制通过 VALID 和 READY 信号确保数据可靠传输，是理解 AXI 协议的基础。让我们直接进入正题！

1. AXI 握手机制简介（1 分钟）

握手机制是 AXI 协议中用于控制数据传输的机制。它通过两个信号实现：

  • VALID：由发送方驱动，表示数据已准备好。
  • READY：由接收方驱动，表示可以接收数据。
    握手机制解决了发送方和接收方速度不一致的问题，确保数据传输的可靠性。

2. VALID 和 READY 信号的作用（2 分钟）

  • VALID 信号

    • 发送方驱动，高电平时表示数据或地址已准备好。
    • 必须保持高电平，直到数据传输完成。

  • READY 信号

    • 接收方驱动，高电平时表示可以接收数据。
    • 可以在 VALID 之前或之后拉高。

只有当 VALID 和 READY 同时为高时，数据才会传输。

3. 握手机制的工作流程（4 分钟）

握手机制的工作流程可以分为以下几个步骤：
6. 发送方准备数据

  • 发送方将数据或地址放置在总线上，并拉高 VALID 信号。

  1. 接收方响应

    • 接收方在准备好接收数据时，拉高 READY 信号。

  2. 数据传输

    • 当 VALID 和 READY 同时为高时，数据从发送方传输到接收方。

  3. 完成握手

    • 数据传输完成后，VALID 和 READY 信号可以拉低，准备下一次传输。

关键点：

  • VALID 和 READY 信号的拉高顺序可以灵活变化。
  • 数据传输只在 VALID 和 READY 同时为高时发生。

5. 总结与预告（1 分钟）

总结一下：

  • 握手机制通过 VALID 和 READY 信号确保数据传输的可靠性。
  • 数据传输发生在 VALID 和 READY 同时为高的时刻。
